# 17-code-review

Lead with actionable findings ordered by severity. Reference files and lines, explain the user-visible or engineering risk, and propose a concrete fix. Prioritize crashes, data loss, security exposure, lifecycle leaks, threading bugs, broken navigation, migration risk, and missing tests around changed behavior. Keep summaries secondary to findings.
