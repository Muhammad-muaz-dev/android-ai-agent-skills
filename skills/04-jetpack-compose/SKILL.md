# 04-jetpack-compose

Use Compose for declarative UI. Keep composables stateless where practical, hoist state to ViewModels or callers, model UI with immutable state, and separate screen, section, and reusable component functions. Prefer Material 3 components, theme tokens, previews for key states, accessibility semantics, and stable keys in lazy lists. Avoid business logic, navigation decisions, and data access inside composables.
